    Crear el directorio app/api/public/slots/ con route.ts.

    ARCHIVO — app/api/public/slots/route.ts:

    IMPORTS: NextRequest de 'next/server', prisma de '@/lib/prisma'.

    CONSTANTE a nivel módulo:
      const BOOKING_WINDOW_DAYS = 30

    FUNCIÓN generateSlots(startTime: string, endTime: string, durationMin: number): string[]:
    - Parsear startTime y endTime como "HH:MM": const [sh, sm] = startTime.split(':').map(Number), y [eh, em] similarmente
    - cur = sh*60 + sm; end = eh*60 + em
    - while (cur + durationMin <= end): generar string HH:MM con padStart(2,'0') y avanzar cur += durationMin
    - Devolver array de strings HH:MM

    FUNCIÓN GET(request: NextRequest):
    1. Leer doctorIdStr = request.nextUrl.searchParams.get('doctorId') y dateStr = request.nextUrl.searchParams.get('date')
    2. Si alguno es null: return Response.json({ error: 'doctorId y date son requeridos' }, { status: 400 })
    3. Parsear doctorId = parseInt(doctorIdStr, 10). Si isNaN: return 400.
    4. Construir targetDate SIN sufijo Z (evita timezone shift per Pitfall 1):
         const [year, month, day] = dateStr.split('-').map(Number)
         const targetDate = new Date(year, month - 1, day)   // constructor LOCAL
    5. Validar ventana de 30 días:
         const today = new Date(); today.setHours(0, 0, 0, 0)
         const maxDate = new Date(today); maxDate.setDate(today.getDate() + BOOKING_WINDOW_DAYS)
         if (targetDate < today || targetDate > maxDate) return Response.json({ slots: [] })
    6. dayOfWeek = targetDate.getDay()  // 0=Domingo, 1=Lunes... 6=Sábado — debe coincidir con convención del seed
    7. Query paralela con Promise.all:
         [doctor, existingAppointments] = await Promise.all([
           prisma.doctor.findUnique({ where: { id: doctorId, isActive: true }, include: { availabilities: { where: { dayOfWeek } } } }),
           prisma.appointment.findMany({ where: { doctorId, date: targetDate, status: { not: 'CANCELLED' } }, select: { time: true } })
         ])
    8. Si !doctor: return Response.json({ error: 'Médico no encontrado' }, { status: 404 })
    9. bookedTimes = new Set(existingAppointments.map(a => a.time))
    10. Generar slots: para cada av en doctor.availabilities, llamar generateSlots(av.startTime, av.endTime, doctor.durationMin) y mapear a { time, available: !bookedTimes.has(time) }
    11. Ordenar por time (localeCompare) y return Response.json({ slots })
    12. Wrappear en try/catch externo: si falla, return Response.json({ error: 'Error al obtener slots' }, { status: 500 })

    CRÍTICO — Prisma date query: targetDate se construye con new Date(year, month-1, day). Cuando se pasa a prisma.appointment.findMany({ where: { date: targetDate } }), Prisma 6 con @db.Date hace comparación de fecha. Si el comportamiento de igualdad no funciona como esperado, ajustar a un rango: { date: { gte: startOfDay, lt: endOfDay } } donde startOfDay = new Date(year, month-1, day) y endOfDay = new Date(year, month-1, day+1).

    VERIFICAR en prisma/seed.ts: si los seeds usan dayOfWeek con 0=Lunes (ISO week) en vez de 0=Domingo (JS), adaptar el cálculo a: dayOfWeek = targetDate.getDay() === 0 ? 7 : targetDate.getDay() — y documentar en un comentario inline qué convención usa el seed.
