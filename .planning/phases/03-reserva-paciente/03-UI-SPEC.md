# Phase 3 — Reserva Paciente UI Design Contract

> Visual and interaction contract for the public 5-step Booking Wizard. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom tokens via `globals.css` + Tailwind v4 `@theme` |
| Preset | not applicable |
| Component library | none (hand-rolled with Tailwind v4 design tokens) |
| Icon library | lucide-react |
| Font | Outfit (headings, `font-display`) + Inter (UI/forms, `font-body`) via `next/font/google` |

Source: `app/globals.css`, `.planning/PROJECT.md`, `02-UI-SPEC.md`

**Glassmorphism is mandatory for this phase.** Unlike the admin panel (which uses flat surfaces), the Booking Wizard applies "Modern Clinical Sanctuary" glassmorphism: `backdrop-blur-md` + `bg-white/80` cards + soft teal borders. This is the visual differentiator between the public and admin layers.

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), chip internal spacing |
| sm | 8px | Compact spacing — chip padding (`px-3 py-2`), calendar cell gap |
| md | 16px | Default element spacing — form field gaps, card inner padding |
| lg | 24px | Section padding — wizard card padding (`p-6`), step header padding |
| xl | 32px | Layout gaps — space between step header and content |
| 2xl | 48px | Vertical rhythm — between wizard card and page edge |
| 3xl | 64px | Page-level top/bottom padding on the `/reservar` canvas |

Exceptions:
- Time slot chips: `px-3 py-2` (8px vertical — multiple of 4). Minimum 44px height to meet touch target requirement.
- Calendar day cells: minimum 40×40px per cell for tap accessibility.
- Step progress indicator dots: `w-3 h-3` (12px) active/completed, `w-2 h-2` (8px) upcoming — size differentiates state, no 6px values used.

## Typography

| Role | Size | Weight | Line Height | Font | Tailwind |
|------|------|--------|-------------|------|---------|
| Body | 16px | 400 (regular) | 1.5 | Inter (`font-body`) | `text-base` |
| Label / Helper | 14px | 400 (regular) | 1.5 | Inter (`font-body`) | `text-sm` |
| Step heading | 20px | 600 (semibold) | 1.2 | Outfit (`font-display`) | `text-xl font-semibold` |
| Page heading | 28px | 600 (semibold) | 1.1 | Outfit (`font-display`) | `text-3xl font-semibold` |

Rules:
- Form inputs: 16px Inter, weight 400. Body-scale input text prevents iOS auto-zoom on focus.
- Card titles (specialty name, doctor name): 16px Inter, weight 600 (`text-base font-semibold`).
- Card subtitles (doctor specialty, duration): 14px Inter, weight 400, `text-text-secondary`.
- Calendar month/year heading: 16px Outfit semibold.
- Calendar day-of-week headers (L M X J V S D): 14px Inter weight 600, uppercase, `text-text-muted`. (migrated from 12px — fits the declared 4-size scale)
- Calendar day numbers: 14px Inter, weight 400 (available) / 400 (unavailable/muted).
- Confirmation summary labels: 14px Inter, weight 600, uppercase, `tracking-wide text-text-muted`. (migrated from 12px — fits the declared 4-size scale)
- Confirmation summary values: 16px Inter, weight 400, `text-text-primary`.
- Step label below progress dots: 14px Inter, weight 400, `text-text-secondary`. (migrated from 12px — fits the declared 4-size scale)
- Only two weights used across the entire phase: 400 (regular) and 600 (semibold).

**Valid type scale: 14px / 16px / 20px / 28px — exactly 4 sizes. No 12px values exist in this phase.**

**Wizard note:** Body text is 16px (vs. 14px in admin) because the wizard targets patients on mobile — larger text reduces friction.

## Color

| Role | Value | Tailwind token | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | `#f8fafc` | `bg-background` | Page canvas behind the wizard card |
| Secondary (30%) | `rgba(255,255,255,0.80)` | `bg-white/80` | Glassmorphic wizard card surface |

Accent (`#14b8a6` / `text-accent` / `#0d9488` / `text-primary`) reserved for:
1. Available calendar day cells — background `bg-primary/20`, text `text-primary` (teal)
2. Selected calendar day cell — background `bg-gradient-to-br from-primary to-accent`, text `text-white`
3. Available time slot chips — `border border-accent/40 text-primary bg-primary/10`, hover `bg-primary/20`
4. Selected time slot chip — `bg-gradient-to-r from-primary to-accent text-white`
5. Step progress indicator — active dot `bg-accent`, connector line `bg-accent`
6. Primary CTA button gradient — `bg-gradient-to-r from-primary to-accent text-white`
7. Focus ring on form inputs — `border-focus` = `rgba(13,148,136,0.4)` + `ring-[rgba(13,148,136,0.2)]`
8. Glassmorphic card border — `border border-accent/20` (the soft teal outline)
9. "Hoy" marker on calendar — small teal dot below the day number
10. Back link / step navigation secondary link — `text-primary hover:text-accent`

Unavailable / occupied states:
- Unavailable calendar days: `text-text-muted`, `cursor-not-allowed`, no background
- Past calendar days: same as unavailable
- Occupied time slots: `bg-slate-100 text-text-muted border border-border cursor-not-allowed opacity-60`

## Layout and Route Structure

**Route:** `/reservar` — public, no auth required, no admin sidebar.

**Route group:** `app/(public)/reservar/page.tsx`. Uses a separate layout `app/(public)/layout.tsx` with no sidebar — full-width page with centered wizard card.

**Page layout:**
- Canvas: `min-h-screen bg-background flex items-start justify-center px-4 py-16` (py-16 = 64px top/bottom)
- Wizard card: `w-full max-w-2xl` — responsive single column, centered
- On mobile (< 640px): full-width with `mx-4` horizontal margin

**State management:** Single `'use client'` component (`BookingWizard`) that owns all wizard state via `useState`. No routing between steps — steps rendered conditionally inside the wizard card. State is lost on reload — intentional per D-decision in CONTEXT.md.

## Screen-by-Screen Interaction Contracts

### Wizard Shell

**Step progress bar:**
- 5 dots in a horizontal row, centered above the step content
- Active step: `w-3 h-3 rounded-full bg-accent` (12px)
- Completed steps: `w-3 h-3 rounded-full bg-primary/40` (12px)
- Upcoming steps: `w-2 h-2 rounded-full bg-border` (8px)
- Connector lines between dots: `flex-1 h-0.5`, completed connector = `bg-primary/40`, upcoming = `bg-border`
- Step label below dots: 14px Inter weight 400 `text-text-secondary` — shows current step name only (e.g. "Especialidad", "Médico", "Fecha y hora", "Tus datos", "Confirmación")

**Navigation pattern:**
- "Volver" (back) link: top-left inside wizard card, `← Volver` — `text-sm text-primary hover:text-accent`, hidden on step 1
- Primary CTA button: bottom-right of each step, full-width on mobile
- No "Siguiente" button on step 1–3 — selection advances the step immediately (auto-advance on click)
- Steps 4–5: explicit "Confirmar turno" / "Volver al inicio" buttons (no auto-advance)

### Step 3: Elegir fecha y hora

**Component type:** Client Component (part of `BookingWizard`)

**Heading:** `<h2>` "Elegí la fecha y hora" — 20px Outfit semibold

**Subtitle:** "Dr./Dra. {selectedDoctor.name}" — 14px Inter `text-text-secondary`

**Layout:** Two sections stacked — calendar above, time slots below.

#### Calendar (monthly grid)

**Container:** `bg-white/60 backdrop-blur-sm rounded-xl border border-border p-4`

**Month navigation:**
- Row: `<` chevron button + "Mayo 2026" (16px Outfit semibold) + `>` chevron button
- Chevron buttons: `ChevronLeft` / `ChevronRight` lucide icons, 20px, `text-text-secondary hover:text-primary`
- Prev-month button disabled if current month is already today's month
- Future navigation capped at +1 month from today (30-day window = at most 2 months visible)

**Day-of-week headers row:** L M X J V S D — 14px Inter weight 600, uppercase, `text-text-muted`, centered per cell

**Day cells grid:** `grid grid-cols-7 gap-1`

Each day cell: `w-10 h-10 flex items-center justify-center rounded-lg text-sm`

| State | Classes |
|-------|---------|
| Available | `bg-primary/10 text-primary font-semibold cursor-pointer hover:bg-primary/20` |
| Selected | `bg-gradient-to-br from-primary to-accent text-white font-semibold shadow-sm` |
| Unavailable / past | `text-text-muted cursor-not-allowed` |
| Today marker | Available or unavailable class + small `2px 2px` teal dot below number (`absolute bottom-1 left-1/2 -translate-x-1/2 w-1 h-1 rounded-full bg-accent`) |
| Empty cells (leading/trailing) | `invisible` (preserve grid alignment) |

Click on available day: sets `selectedDate`, triggers slot fetch, scrolls to time slot section.

**Booking window:** 30 days from today. Days beyond this window: unavailable style.

#### Time Slots

**Section heading:** "Horarios disponibles" — 14px Inter weight 600, `text-text-secondary`, `uppercase tracking-wide`

**Shown only after a day is selected.** Before selection: placeholder text "Seleccioná un día para ver los horarios" — 14px Inter `text-text-muted`, centered.

**Loading state (slot fetch):** 8 skeleton chips — `bg-slate-200 animate-pulse rounded-full h-10 w-20`

**Chip grid:** `grid grid-cols-4 gap-2 sm:grid-cols-5`

Each chip: `rounded-full px-3 py-2 text-sm font-semibold min-h-[44px] flex items-center justify-center transition-all duration-150`

| State | Classes |
|-------|---------|
| Available | `border border-accent/40 text-primary bg-primary/10 cursor-pointer hover:bg-primary/20` |
| Selected | `bg-gradient-to-r from-primary to-accent text-white shadow-sm` |
| Occupied (visible, not clickable) | `bg-slate-100 text-text-muted border border-border cursor-not-allowed opacity-60` |

Selected chip + available day: activates the primary CTA button "Confirmar horario".

**"No availability" state (doctor has no slots in 30-day window):**

Shown after day selection returns zero slots for ALL days, or surfaced at step entry if API confirms zero availability.

- Icon: `CalendarX` from lucide-react, 40px, `text-text-muted`
- Heading: "Este médico no tiene turnos disponibles" — 16px Outfit semibold
- Body: "No hay horarios disponibles en los próximos 30 días. Podés elegir otro médico." — 14px Inter `text-text-secondary`
- CTA: "← Elegir otro médico" — secondary button, returns to step 2

**Step 3 primary CTA:** "Confirmar horario" — primary button, enabled only when a date AND time slot are selected. Full-width on mobile.

**Form fields:**

| Email — invalid format | "Ingresá un email válido." |

**Primary CTA:** "Confirmar turno" — primary button, full-width. Shows `Loader2 animate-spin` during submit. Disabled during submission.

**Race condition error (slot taken):** Shown inline below the CTA button after failed POST.

## API Surface (UI-relevant behavior)

These are read-only for the UI contract — implementation lives in plans. Listed here so the executor understands what triggers loading states.

| Endpoint | Trigger | Loading State |
|----------|---------|---------------|
| `GET /api/public/specialties` | Step 1 mount | 6 skeleton cards |
| `GET /api/public/doctors?specialtyId=` | Step 2 mount (after specialty selected) | 3 skeleton rows |
| `GET /api/public/slots?doctorId=&date=` | Day cell click in step 3 | 8 skeleton chips |
| `POST /api/public/appointments` | "Confirmar turno" button | Button loading state + disabled |

Error boundary: if any GET fails, show inline error (`text-sm text-error`) within the step area + retry link "Intentá nuevamente" (re-triggers the fetch).

## Accessibility Notes

- All clickable cards: `role="button"` + `tabIndex={0}` + `onKeyDown` handling for Enter/Space
- Calendar day cells: `aria-label="15 de mayo, disponible"` / `"15 de mayo, no disponible"` / `"15 de mayo, seleccionado"`
- Time slot chips: `aria-label="10:30, disponible"` / `"10:30, ocupado"` / `"10:30, seleccionado"`
