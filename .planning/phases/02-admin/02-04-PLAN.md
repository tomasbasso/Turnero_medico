    Create app/api/admin/doctors/route.ts. Exports GET and POST.

    Imports: NextRequest from 'next/server'; verifyToken and COOKIE_NAME from '@/lib/auth'; requireAdmin from '@/lib/auth-helpers'; prisma from '@/lib/prisma'.

    GET handler — authenticated (any valid role can read):
    - Signature: export async function GET(request: NextRequest)
    - Auth: const token = request.cookies.get(COOKIE_NAME)?.value; const payload = token ? verifyToken(token) : null; if (!payload) return Response.json({ error: 'No autenticado' }, { status: 401 })
    - Query with specialty: const doctors = await prisma.doctor.findMany({ where: { isActive: true }, orderBy: { name: 'asc' }, include: { specialty: { select: { id: true, name: true, color: true } } } })
    - Return: Response.json({ doctors })

    POST handler — ADMIN only, creates doctor WITHOUT avatar:
    - Signature: export async function POST(request: NextRequest)
    - Auth: const authResult = requireAdmin(request); if (authResult instanceof Response) return authResult
    - Body parse: const body = await request.json().catch(() => null)
    - Validate required fields: if (!body?.name || !body?.specialtyId) return Response.json({ error: 'Nombre y especialidad son requeridos' }, { status: 400 })
    - Explicit destructure only allowed fields (NEVER spread body): const { name, specialtyId, bio, phone, durationMin } = body
    - Create without avatar: const doctor = await prisma.doctor.create({ data: { name: name.trim(), specialtyId: Number(specialtyId), bio: bio ?? null, phone: phone ?? null, durationMin: Number(durationMin ?? 30) } })
    - Return: Response.json({ doctor }, { status: 201 })

    The doctor ID returned in the response is used by the client to upload avatar then call PATCH. This two-step flow means POST does not handle avatar — the PUT handler on [id] accepts the avatar field.

    NEVER use NextResponse.json(). NEVER spread body into Prisma data.
  </action>

    - Return: Response.json({ ok: true })

    FILE B: app/api/admin/doctors/[id]/availability/route.ts — exports GET and PUT.

    Imports: NextRequest from 'next/server'; verifyToken and COOKIE_NAME from '@/lib/auth'; requireAdmin from '@/lib/auth-helpers'; prisma from '@/lib/prisma'.

    GET handler — authenticated (any role can read availability):
    - Signature: export async function GET(request: NextRequest, { params }: { params: Promise<{ id: string }> })
    - Auth: cookie check + verifyToken, return 401 if invalid
    - const { id } = await params; const doctorId = parseInt(id, 10)
    - Query: const availabilities = await prisma.availability.findMany({ where: { doctorId }, orderBy: { dayOfWeek: 'asc' } })
    - Return: Response.json({ availabilities })

    PUT handler — ADMIN only, full replace in a transaction:
    - Auth: requireAdmin check
    - const { id } = await params; const doctorId = parseInt(id, 10); if (isNaN(doctorId)) return 400
    - Body parse: const body = await request.json().catch(() => null)
    - Validate body.availabilities is an array: if (!Array.isArray(body?.availabilities)) return Response.json({ error: 'availabilities debe ser un array' }, { status: 400 })
    - Server-side validation: for each item in body.availabilities, validate endTime > startTime. If any invalid: return Response.json({ error: 'La hora de fin debe ser posterior a la hora de inicio' }, { status: 400 })
    - Transaction: await prisma.$transaction([ prisma.availability.deleteMany({ where: { doctorId } }), prisma.availability.createMany({ data: body.availabilities.map((a: { dayOfWeek: number; startTime: string; endTime: string }) => ({ doctorId, dayOfWeek: a.dayOfWeek, startTime: a.startTime, endTime: a.endTime })) }) ])
    - Return: Response.json({ ok: true })

    NEVER spread body into Prisma. NEVER use NextResponse.json(). Always await params.
