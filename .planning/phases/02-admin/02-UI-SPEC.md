# Phase 2 — Admin Panel UI Design Contract

> Visual and interaction contract for frontend phases. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom tokens via `globals.css` + Tailwind v4 `@theme` |
| Preset | not applicable |
| Component library | none (hand-rolled with Tailwind v4 design tokens) |
| Icon library | lucide-react |
| Font | Outfit (headings, `font-display`) + Inter (UI/forms, `font-body`) via `next/font/google` |

Source: `app/globals.css`, `scratch/DESIGN.md`, `.planning/PROJECT.md`

---

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), inline badge padding |
| sm | 8px | Compact element spacing — nav item padding, chip padding (`px-2 py-1`) |
| md | 16px | Default element spacing — form field gaps, card inner padding |
| lg | 24px | Section padding — page content padding (`p-6`), card padding |
| xl | 32px | Layout gaps — space between page header and content |
| 2xl | 48px | Major section breaks — between dashboard cards and next section |
| 3xl | 64px | Page-level spacing — not used in admin (sidebar layout fills viewport) |

Exceptions:
- Nav items in sidebar: `px-3 py-3` (12px vertical — multiple of 4). Current `AdminSidebar.tsx` uses `py-2.5`; this spec declares `py-3` and the correction will be applied when the sidebar is updated in this phase.
- Avatar touch target / upload button: minimum 44px height
- Time-range rows in availability editor: minimum 40px height for tap accessibility

---

## Typography

| Role | Size | Weight | Line Height | Font | Tailwind |
|------|------|--------|-------------|------|---------|
| Body | 14px | 400 (regular) | 1.5 | Inter (`font-body`) | `text-sm` |
| Label | 12px | 400 (regular) | 1.4 | Inter (`font-body`) | `text-xs` |
| Heading (page) | 20px | 600 (semibold) | 1.2 | Outfit (`font-display`) | `text-xl font-semibold` |
| Display (stat card number) | 28px | 600 (semibold) | 1.1 | Outfit (`font-display`) | `text-3xl font-semibold` |

Rules:
- Form inputs: 14px Inter, weight 400.
- Table/list content: 14px Inter, weight 400.
- Section sub-headings (e.g. day names in availability editor): 12px Inter, weight 600 (`text-xs font-semibold uppercase tracking-wide`).
- Page title: 20px Outfit semibold — placed in a `<h1>` inside each page's content header area.
- Only two weights are used across the entire phase: 400 (regular) and 600 (semibold). Weight 500 (medium) and 700 (bold) are not used. At 28px, semibold (600) reads as visually dominant without requiring bold.

## Color

| Role | Value | Tailwind token | Usage |
|------|-------|---------------|-------|
| Dominant (60%) | `#f8fafc` | `bg-background` | Main content area canvas |
| Secondary (30%) | `#0f172a` (sidebar) + `#ffffff` (cards) | `bg-slate-900` / `bg-surface` | Sidebar background + all cards/drawers |
| Accent (10%) | `#14b8a6` | `text-accent` | See reserved list below |
| Destructive | `#ef4444` | `text-error` / `bg-error` | Eliminar buttons, delete confirmation text |

Accent (`#14b8a6` / `text-accent`) reserved for:
1. Active nav item text in sidebar (`bg-primary/20 text-accent`)
2. Focus ring on form inputs (`border-focus` = `rgba(13,148,136,0.4)`)
3. Primary button gradient endpoint (`from-primary to-accent`)
4. Stat card "Confirmados" count number
5. Drawer sheet header accent bar (4px left border on the active/open drawer)

Status colors (from `lib/utils.ts` `STATUS_COLORS` — use exactly as defined):
- `PENDING` → `bg-amber-100 text-amber-700` — Dashboard card accent + chip
- `CONFIRMED` → `bg-teal-100 text-teal-700` — Dashboard card accent + chip
- `COMPLETED` → `bg-slate-100 text-slate-600` — Dashboard card accent + chip
- `CANCELLED` → `bg-red-100 text-red-700` — Dashboard card accent + chip

Specialty color picker: stored in `Specialty.color` (default `#0d9488`). Shown as a 20×20px filled circle swatch inline in the list row and as a `<input type="color">` in the form.

## Surfaces and Elevation

| Surface | Background | Shadow | Radius |
|---------|-----------|--------|--------|
| Page canvas | `bg-background` (#f8fafc) | none | — |
| Card (stat, list container) | `bg-surface` (#ffffff) | `shadow-card` (`0 4px 6px -1px rgba(15,23,42,0.04)`) | `rounded-xl` (12px) |
| Sidebar | `bg-slate-900` | none | — |
| Drawer/Sheet overlay | `bg-surface` (#ffffff) | `shadow-modal` (`0 20px 25px -5px rgba(13,148,136,0.08)`) | `rounded-l-xl` (left-attached) |
| Form inputs | `bg-white` | none | `rounded-lg` (8px) |
| Status chips / specialty chips | per STATUS_COLORS | none | `rounded-full` (999px) |
| Avatar fallback circle | `bg-primary/20` with `text-primary` initials | none | `rounded-full` |

Border rule (No Harsh Borders): inputs and cards use `border border-border` where `--border: #e2e8f0`. No `border-slate-900` or `border-black`. Focus state: `focus:border-[rgba(13,148,136,0.4)] focus:ring-2 focus:ring-[rgba(13,148,136,0.2)]`.

## Screen-by-Screen Interaction Contracts

### 1. Dashboard (`/admin`)

**Component type:** Server Component (no client interactivity)

**Layout:** `p-6` page padding. Title row: `<h1>` "Dashboard" + subtitle "Semana del {fecha lunes} al {fecha domingo}".

**Focal point:** The 4 stat card count numbers (28px Outfit semibold) are the primary focal point — the eye lands here first before scanning the labels below them.

**Stat cards grid:** `grid grid-cols-2 gap-4` on mobile, `grid-cols-4` on md+. Each card:
- Background: `bg-surface rounded-xl shadow-card p-6`
- Top row: status label (12px Inter weight 400, status text color) + status icon (lucide, 16px)
- Number: 28px Outfit semibold, `text-text-primary`
- Sub-label: "turnos esta semana" — 12px Inter, `text-text-secondary`
- Left accent border: 4px solid, color per status:
  - PENDING → `border-l-4 border-amber-400`
  - CONFIRMED → `border-l-4 border-teal-500`
  - COMPLETED → `border-l-4 border-slate-400`
  - CANCELLED → `border-l-4 border-red-400`

**Loading state:** None (D-04: Server Component, no skeleton).

**Empty state:** Not applicable — counts display as 0 when no appointments exist.

---

### 2. Specialties List (`/admin/especialidades`)

**Component type:** Page = Server Component. Drawer form = Client Component (`'use client'`).

**Layout:** `p-6`. Page header row: `<h1>` "Especialidades" (left) + "Crear especialidad" button (right, primary style, ADMIN only).

**List:** `bg-surface rounded-xl shadow-card` container. Each row:
- Color swatch: 20×20px filled circle (`style={{ backgroundColor: specialty.color }}`, `rounded-full`)
- Name: 14px Inter semibold `text-text-primary`
- Description: 14px Inter `text-text-secondary` (truncated to 1 line)
- Actions (ADMIN only, right-aligned): "Editar" icon button (`aria-label="Editar especialidad"`) + "Eliminar" destructive icon button (`aria-label="Eliminar especialidad"`, `text-error hover:text-error`)
- Divider between rows: `border-b border-border` (ghost rule compliant — `#e2e8f0`)

**RECEPTIONIST view:** Action column hidden entirely. No create button. Read-only banner NOT required — absence of controls is sufficient.

**Empty state:** Centered inside the list container.
- Heading: "Sin especialidades"
- Body: "Aún no hay especialidades registradas. Creá la primera para empezar."
- CTA button: "Crear especialidad" (only if ADMIN)

---

### 3. Specialty Create/Edit Form (Drawer)

**Trigger:** "Crear especialidad" button or "Editar" row action opens a right-side drawer/sheet overlay. No navigation. URL does NOT change.

**Drawer anatomy:**
- Overlay backdrop: `bg-black/40` full-screen, click to dismiss
- Sheet panel: fixed right edge, `w-full max-w-md`, `bg-surface rounded-l-xl shadow-modal`, slides in from right (framer-motion: `x: 400 → 0`, `duration: 0.25s ease-out`)
- Header: "Crear especialidad" or "Editar especialidad" — 20px Outfit semibold + `×` close button (top right, `text-text-secondary`)
- Accent bar: 4px `bg-gradient-to-b from-primary to-accent` left border strip on drawer panel

**Form fields (stacked, `gap-4`):**

| Field | Type | Label | Validation |
|-------|------|-------|-----------|
| Nombre | `<input type="text">` | "Nombre" | Required, max 100 chars |
| Descripción | `<textarea rows={3}>` | "Descripción" | Optional, max 500 chars |
| Color | `<input type="color">` | "Color" | Default `#0d9488`; show swatch preview 24×24px beside picker |

**Input style:** `w-full rounded-lg border border-border bg-white px-3 py-2 text-sm text-text-primary focus:outline-none focus:border-[rgba(13,148,136,0.4)] focus:ring-2 focus:ring-[rgba(13,148,136,0.2)]`

**Label style:** `block text-xs text-text-secondary mb-1`

**Footer (sticky bottom of drawer):** Secondary dismiss control "Descartar cambios" (secondary button — see note below) + "Guardar especialidad" (primary button). During submit: "Guardar especialidad" shows spinner + disabled state.

**Note on dismiss control:** "Descartar cambios" is a secondary dismiss control, not a primary CTA. It is exempt from the CTA noun-qualifier rule — its role is modal dismissal, not a data action.

**Success:** Drawer closes, list re-fetches (router.refresh()), no toast needed.

**Error:** Inline error text below the submit button — `text-xs text-error`. Copy: "Ocurrió un error al guardar. Intentá nuevamente."

---

### 4. Delete Specialty — Confirmation

**Pattern:** Inline confirmation within the row (not a modal dialog). On first "Eliminar" click, the row action area transforms to: "¿Confirmar eliminación?" text (12px `text-text-secondary`) + "Sí, eliminar" (`text-error font-semibold text-sm`) + "Descartar cambios" (`text-text-secondary text-sm`). No dialog overlay.

**Destructive action label:** "Sí, eliminar"

---

### 5. Doctors List (`/admin/medicos`)

**Component type:** Page = Server Component. Drawer form = Client Component.

**Layout:** `p-6`. Header: `<h1>` "Médicos" + "Crear médico" button (ADMIN only).

**List:** `bg-surface rounded-xl shadow-card`. Each row:
- Avatar: 40×40px circle. If `doctor.avatarUrl` → `<img>` with `rounded-full object-cover`. Else → initials via `getInitials(name)`, `bg-primary/20 text-primary text-sm font-semibold rounded-full flex items-center justify-center`
- Name: 14px Inter semibold `text-text-primary`
- Specialty chip: `bg-teal-100 text-teal-700 rounded-full px-2 py-0.5 text-xs` with specialty color dot (8×8px circle in `specialty.color`) prepended
- Phone: 14px Inter `text-text-secondary`
- Duration: "XX min" — 12px Inter `text-text-muted`
- Actions (ADMIN only): "Editar" icon button (`aria-label="Editar médico"`) + "Disponibilidad" link button (`text-accent`) + "Eliminar" destructive icon button (`aria-label="Eliminar médico"`)
- Divider: `border-b border-border`

**RECEPTIONIST:** Actions column hidden. "Disponibilidad" link visible (read-only view of availability).

**Empty state:**
- Heading: "Sin médicos"
- Body: "Aún no hay médicos registrados. Agregá el primero para comenzar."
- CTA (ADMIN only): "Crear médico"

---

### 6. Doctor Create/Edit Form (Drawer)

**Same drawer anatomy as Specialty drawer** (width `max-w-lg` — wider to accommodate avatar upload).

**Form fields (stacked, `gap-4`):**

| Field | Type | Label | Validation |
|-------|------|-------|-----------|
| Nombre | `<input type="text">` | "Nombre completo" | Required, max 120 chars |
| Especialidad | `<select>` | "Especialidad" | Required; options from specialties list |
| Teléfono | `<input type="tel">` | "Teléfono" | Optional, max 20 chars |
| Duración del turno | `<input type="number" min="10" max="120" step="5">` | "Duración del turno (minutos)" | Required; default 30 |
| Biografía | `<textarea rows={4}>` | "Biografía" | Optional, max 1000 chars |
| Avatar | `<input type="file" accept="image/*">` | "Foto de perfil" | Optional, max 2 MB |

**Avatar upload UX:**
- Hidden `<input type="file">` triggered by a styled "Subir foto" button
- Preview: 64×64px circle thumbnail shown immediately on file select (via `URL.createObjectURL`)
- Upload happens on form submit via `uploadAvatar(file, doctorId)` from `lib/supabase.ts`
- If no file selected: show initials avatar fallback (64×64px, `bg-primary/20`)
- Error: "La imagen no pudo subirse. Intentá con otro archivo." below the avatar area

**Select style:** Same border/radius/focus as text input. Arrow icon via `appearance-none` + background SVG.

**Footer:** Same pattern as Specialty drawer — "Descartar cambios" (secondary dismiss control) + "Guardar médico" (primary button) with loading state.

### 7. Delete Doctor — Confirmation

Same inline row confirmation pattern as specialties.

**Destructive action label:** "Sí, eliminar"

**Confirmation copy:** "¿Confirmar eliminación? Esta acción no se puede deshacer."

### 8. Availability Editor (`/admin/medicos/[id]/disponibilidad`)

**Component type:** Separate page (not drawer — complexity warrants own route). Server Component fetches existing availability; Client Component handles add/remove interactions.

**Layout:** `p-6`. Header: `<h1>` "Disponibilidad — {doctor.name}"` + back link "← Volver a Médicos" (14px `text-accent`).

**Structure:** 7 day sections (lunes to domingo). Day label order: Lunes, Martes, Miércoles, Jueves, Viernes, Sábado, Domingo.

**Each day section:**
- Day heading: `text-xs font-semibold uppercase tracking-wide text-text-secondary` (e.g. "LUNES")
- Time range rows: each row = `startTime` input (type="time") + "—" separator + `endTime` input (type="time") + "×" remove button (`text-error`)
- "Agregar franja" button: `+ Agregar franja` — secondary/ghost style, `text-sm text-accent`, below the ranges for that day
- Empty day (0 ranges): shows "Sin disponibilidad" in `text-text-muted text-xs` before the add button

**Time input style:** `rounded-lg border border-border px-2 py-1.5 text-sm text-text-primary w-28` with same focus style.

**Validation:** `endTime` must be > `startTime`. If invalid: `text-xs text-error` "La hora de fin debe ser posterior a la hora de inicio." shown inline below the row.

**Save:** Single "Guardar disponibilidad" primary button at the bottom of the page. Replaces entire availability for the doctor (full replace, not delta). Loading state on button during save. Success: "Disponibilidad guardada correctamente" — inline success message below button (`text-sm text-success`). Error: `text-sm text-error` "No se pudo guardar. Intentá nuevamente."

**RECEPTIONIST view:** All inputs disabled (`disabled` attribute, visual: `opacity-60 cursor-not-allowed`). "Agregar franja" and "×" buttons hidden. "Guardar disponibilidad" button hidden.

## Buttons

| Variant | Background | Text | Border | Hover |
|---------|-----------|------|--------|-------|
| Primary | `bg-gradient-to-r from-primary to-accent` | `text-white` | none | `shadow-[0_0_12px_rgba(20,184,166,0.4)]` |
| Secondary | `bg-transparent` | `text-primary` | `border border-[rgba(13,148,136,0.3)]` | `bg-primary/5` |
| Destructive | `bg-transparent` | `text-error` | none | `bg-error/5` |
| Ghost | `bg-transparent` | `text-text-secondary` | none | `bg-slate-100` |

All buttons: `rounded-lg` (8px), `px-4 py-2`, `text-sm font-semibold`, `transition-all duration-150`. Minimum touch target: 44px height for primary/destructive actions.

Disabled state (all variants): `opacity-50 cursor-not-allowed pointer-events-none`.

Loading state: spinner icon (lucide `Loader2` with `animate-spin`, 16px) replaces leading icon or prepends to label. Button remains same size to prevent layout shift.
