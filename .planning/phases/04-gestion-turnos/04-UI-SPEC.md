# Phase 4 — UI Design Contract: Gestión de Turnos

> Visual and interaction contract for the appointment management admin view.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none — custom design tokens via CSS variables in `app/globals.css` |
| Preset | not applicable |
| Component library | none (custom components matching existing admin patterns) |

## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps (`gap-1`), badge inner padding |
| sm | 8px | Compact element spacing, button icon padding (`p-2`) |
| md | 16px | Default cell padding, filter row gaps (`px-4 py-2`) |
| lg | 24px | Section padding (`p-6` on page wrapper, matches `especialidades/page.tsx`) |
| xl | 32px | Gap between filter bar and table |
| 2xl | 48px | Empty state vertical padding (`py-12`) |
| 3xl | 64px | Not used in this phase |

Exceptions:

- Table cell padding: `px-6 py-4` (horizontal 24px, vertical 16px) — matches `SpecialtiesList` row pattern.
- Touch targets on action buttons: minimum 32px height (`p-1` + 16px icon = ~24px); wrap in `min-h-[32px]` if needed.
- Filter bar inputs: `px-3 py-2` (12px / 8px) — compact for toolbar placement.

## Typography

| Role | Size | Weight | Line Height | Font | Token / Class |
|------|------|--------|-------------|------|---------------|
| Body | 14px (`text-sm`) | 400 regular | 1.5 | Inter | `text-sm text-text-primary` |
| Label / secondary | 12px (`text-xs`) | 400 regular | 1.4 | Inter | `text-xs text-text-secondary` |
| Table header | 12px (`text-xs`) | 600 semibold | 1.4 | Inter | `text-xs font-semibold text-text-muted uppercase tracking-wide` |
| Heading (page title) | 20px (`text-xl`) | 600 semibold | 1.2 | Outfit | `font-display text-xl font-semibold text-text-primary` |

Rules:
- Exactly 2 weights used: 400 (regular) and 600 (semibold).
- Patient name in the Paciente+DNI cell: `text-sm font-semibold text-text-primary`.
- DNI below name: `text-xs text-text-secondary` — use `formatDni()` from `lib/utils.ts`.
- Doctor name in cell: `text-sm text-text-primary`.
- Date+time in cell: date `text-sm text-text-primary`, time below `text-xs text-text-secondary` — use `formatDate()` and `formatTime()`.

## Color

| Role | Value | Token | Usage |
|------|-------|-------|-------|
| Dominant (60%) | #f8fafc | `bg-background` | Page background, filter bar background |
| Secondary (30%) | #ffffff | `bg-surface` | Table card wrapper (`rounded-xl shadow-card`), drawer |
| Accent (10%) | #14b8a6 | `text-accent` / `bg-primary/20` | See reserved list below |
| Destructive | #ef4444 | `text-error` | Cancel action button, destructive confirmation text only |

Accent (`text-accent` / `#14b8a6`) reserved for:
1. The `MessageCircleCheck` icon when `whatsappSent === true` — `text-accent` (teal)
2. The WhatsApp action link/button in the Acciones column when status is CONFIRMED
3. Active sidebar nav item (already established — `text-accent bg-primary/20`)
4. Primary CTA button gradient: `bg-gradient-to-r from-primary to-accent` (matches existing buttons)

Status badge colors (from `STATUS_COLORS` in `lib/utils.ts` — do not deviate):
- PENDING: `bg-amber-100 text-amber-700`
- CONFIRMED: `bg-teal-100 text-teal-700`
- CANCELLED: `bg-red-100 text-red-700`
- COMPLETED: `bg-slate-100 text-slate-600`

Border rule — No Harsh Borders: use `border-border` (`#e2e8f0`) for all dividers. Never use
a border darker than 40% opacity. Table row dividers: `border-b border-border last:border-b-0`
(identical to `SpecialtiesList` pattern).

Left accent bar on StatCard pattern: use `border-l-4` with status-specific border color only
on StatCards (already in Phase 2). Do NOT apply colored left borders to table rows.

## Component Inventory

### Page shell (`app/(admin)/admin/turnos/page.tsx`)

Server Component. Fetches initial appointments for today. Passes data to `AppointmentsList`
client component. Pattern identical to `especialidades/page.tsx`:

```
<div className="p-6">
  <div className="mb-6">
    <h1 className="font-display text-xl font-semibold text-text-primary">Turnos</h1>
  </div>
  <AppointmentsList initialData={...} doctors={...} role={role} />
</div>
```

### Filter bar (`AppointmentsFilterBar`)

Rendered inside `AppointmentsList`. Horizontal row of 4 controls:

| Control | Type | Width | Default |
|---------|------|-------|---------|
| Fecha | `<input type="date">` | `w-40` | today (ISO string) |
| Médico | `<select>` | `w-48` | `""` (all) |
| Estado | `<select>` | `w-40` | `""` (all) |
| DNI | `<input type="text">` | `w-40` | `""` |

Filter bar container: `flex flex-wrap items-center gap-3 mb-6`.

Input/select base class:
```
rounded-lg border border-border bg-surface px-3 py-2 text-sm text-text-primary
focus:border-[rgba(13,148,136,0.4)] focus:outline-none focus:ring-1 focus:ring-[rgba(13,148,136,0.4)]
```

All filtering is client-side. On filter change, component calls `GET /api/admin/appointments`
with query params and re-renders the table without page reload.

### Table (`AppointmentsTable`)

Container: `bg-surface rounded-xl shadow-card` — matches SpecialtiesList wrapper.

Table: `w-full table-auto`.

`<thead>`: `border-b border-border`. Each `<th>`: `px-6 py-3 text-left text-xs font-semibold text-text-muted uppercase tracking-wide`.

`<tbody>`: each `<tr>` has `border-b border-border last:border-b-0 hover:bg-background transition-colors`.

Column layout:

| # | Column | Content | Width |
|---|--------|---------|-------|
| 1 | Paciente + DNI | Name `font-semibold` + DNI `text-xs text-text-secondary` below | auto |
| 2 | Médico | Doctor name | auto |
| 3 | Fecha + Hora | Date + time `text-xs text-text-secondary` below | auto |
| 4 | Estado | Badge pill (STATUS_COLORS) | `w-32` |
| 5 | Acciones | Contextual buttons | `w-40 text-right` |

Each `<td>`: `px-6 py-4 align-top`.

### Status badge

```
<span className={cn('inline-flex items-center rounded-full px-2.5 py-0.5 text-xs font-semibold', STATUS_COLORS[status])}>
  {STATUS_LABELS[status]}
</span>
```

Radius: `rounded-full` (`--radius-full: 9999px`).

### Acciones column — contextual buttons per status

**PENDING:**
- "Confirmar" button: `text-sm font-semibold text-teal-700 hover:text-teal-900 transition-colors`
- "Cancelar" button: `text-sm font-semibold text-error hover:text-red-700 transition-colors`
- Buttons separated by `gap-2` in a `flex items-center` container.
- Confirming triggers `PATCH /api/admin/appointments/[id]` with `{ status: "CONFIRMED" }`.
- After confirm: row updates to CONFIRMED state. WhatsApp link appears.

**CONFIRMED:**
- WhatsApp link: `<a href="https://wa.me/549{phone}?text={encoded}" target="_blank">`.
  Render as `<MessageCircle className="h-4 w-4 text-accent" />` + label "Enviar WA" in `text-sm text-accent`.
- If `whatsappSent === true`: replace icon with `<MessageCircleCheck className="h-4 w-4 text-accent" />` (no label change needed, or label "WA enviado").
- Clicking the link fires `PATCH /api/admin/appointments/[id]/whatsapp` automatically (optimistic — update local state immediately, fire fetch without await blocking UI).

**CANCELLED:**
- No action buttons. Show `<span className="text-xs text-text-muted">—</span>`.

**COMPLETED:**
- No action buttons. Show `<span className="text-xs text-text-muted">—</span>`.

Optional detail icon (Claude's discretion — implement if planner decides to add Drawer):
- `<Eye className="h-4 w-4 text-text-secondary hover:text-text-primary" />` at the end of the Acciones cell.
- Opens `Drawer` (existing `components/admin/Drawer.tsx`) with full appointment detail: phone, email, obra social, notes.

### State change — PENDING inline confirm

Do NOT use a modal for PENDING → CONFIRMED or PENDING → CANCELLED. Use inline optimistic
update with immediate API call. Confirmation for CANCEL only: expand the row or show inline
text `"¿Cancelar este turno?"` with `"Sí, cancelar"` (error color) and `"No"` (secondary) —
identical pattern to `SpecialtiesList` delete confirm.

### Empty state

When no appointments match the active filters:

```
<div className="flex flex-col items-center justify-center py-12 text-center">
  <Calendar className="h-10 w-10 text-text-muted mb-3" />
  <p className="text-sm font-semibold text-text-primary mb-1">Sin turnos para este filtro</p>
  <p className="text-sm text-text-secondary">Probá con otra fecha, médico o estado.</p>
</div>
```

### Loading state

While fetching on filter change: render the table skeleton as 5 rows with
`animate-pulse bg-border rounded h-4` cells — or a simple `<div className="py-8 text-center text-sm text-text-secondary">Cargando...</div>`.
Do not use a full-page spinner; the filter bar must remain visible.

## WhatsApp Integration Detail

Link format:
```
https://wa.me/549{patientPhone}?text={encodeURIComponent(message)}
```

Country prefix: Argentina `54`, mobile prefix `9`. Strip leading `0` from local number if present.

`message` template (use `formatDate` and `formatTime` from `lib/utils.ts`):
```
Hola {patientName}, tu turno con {doctorName} el {formatDate(date)} a las {formatTime(time)} ha sido CONFIRMADO.
```

`whatsappSent` tracking: on link click, fire `PATCH /api/admin/appointments/[id]/whatsapp` with
`{ whatsappSent: true }`. Update local state optimistically — do not block the wa.me link open.

Visual indicator when `whatsappSent === true`:
- Icon: `<MessageCircleCheck className="h-4 w-4 text-accent" />` (lucide-react)
- Color: `text-accent` (#14b8a6 teal)

## API Surface (for executor reference)

| Method | Route | Purpose |
|--------|-------|---------|
| GET | `/api/admin/appointments` | List — accepts query params: `date`, `doctorId`, `status`, `dni` |
| PATCH | `/api/admin/appointments/[id]` | Update status — body: `{ status: "CONFIRMED" \| "CANCELLED" \| "COMPLETED" }` |
| PATCH | `/api/admin/appointments/[id]/whatsapp` | Mark whatsappSent — body: `{ whatsappSent: true }` |

All routes: `requireAdmin()` guard, `Response.json()` (not `NextResponse.json()`), `await params`.
